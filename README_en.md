# postgrad-software-engineering 🎓

Hello reader, welcome to my monorepository for my postgraduate specialization in **Software Engineering** offered by **PUC-Rio**.

Here you will find a collection of exercises and studies organized by subjects and concepts.

The language chosen by the course is **Python**, with content written in `.ipynb` notebooks, so you will not find any other programming language here.

The content is divided into 4 Sprints. Each Sprint has a final project that will be evaluated. Most likely, these projects will be available in separate repositories.

---

## 🗂 Structure

### Sprint 1 — Full Stack Development

**Object-Oriented Programming**

* OOP pillars: abstraction, encapsulation, inheritance, and polymorphism
* Interfaces, abstract classes, and SOLID principles
* Clean Code, style guides, and best practices

**Relational Databases (SQL / PostgreSQL)**

* Conceptual, logical, and physical modeling
* SQL language and database administration
* Non-conventional and distributed databases

**Web Development — Client/Server, APIs, MVC, HTML/CSS/JS, React**

* Web architecture, separation of concerns, and MVC pattern
* HTTP protocol and Python API implementation
* Front-end with HTML, CSS, JavaScript, React, and market frameworks
* Testing, staging, and deployment environments

---

### Sprint 2 — Software Architecture

**Software Architecture and Design**

* Architectural styles and patterns; documentation and architectural views
* GRASP, SOLID principles, and GoF design patterns
* Quality attributes, code smells, and refactoring

**Microservices Architecture**

* SOA, SOAP, and RESTful services
* Microservices (MSA): concepts, implementation, and data management
* Deployment with containers, Docker, and GitHub

**Cloud Architecture and DevOps**

* Deployment pipelines, IaaS/SaaS/PaaS, and continuous integration
* Automated testing, low-risk releases, and telemetry
* DevOps practices: code review, A/B testing, and change management

---

### Sprint 3 — Agile Project and Product Management

**Agile Project Management**

* Agility, BizDev, and DevOps
* Scrum: roles, ceremonies, and artifacts
* Lean, Kanban, and SAFe (Scaled Agile Framework)

**Requirements Engineering and Product Management**

* Requirement types, elicitation, user stories, and acceptance criteria
* Definition of Ready / Done and backlog management
* Product strategy, discovery, and launch

**Interface Design and Prototyping**

* UI elements, wireframes, and low/high-fidelity prototypes
* Cognitive principles, accessibility, and universal design

---

### Sprint 4 — Software Testing and Quality

**Software Testing and Quality**

* Functional, structural, and defect-based testing techniques
* Test automation, Design by Contract, and metamorphic testing
* Process quality models: CMMI and MPS.BR

**Secure Software Development**

* Security in the software lifecycle (OWASP SKF and WSTG)
* Requirements and secure design principles
* Input validation, secure processing, and checklists

**Intelligent Systems Engineering**

* Data Science, Machine Learning, and agile specification of intelligent systems
* Classification and regression algorithms (Decision Trees, KNN, SVM, Ensembles)
* Automated testing and ML model deployment
